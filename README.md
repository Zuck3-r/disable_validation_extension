# フォームバリデーション無効化拡張機能

## 概要
このChrome拡張機能は、ウェブページ上のクライアントサイドのフォームバリデーションを無効化します。バリデーション属性、イベントリスナー、およびクラスベースのバリデーションを削除し、自由にフォームを送信できるようにします。

## 特徴
- `required`、`pattern`、`min`、`max` 属性を削除
- フォームに `novalidate` を設定し、ブラウザのバリデーションを無効化
- バリデーション関連のクラス（例: `chkrequired`、`chkAccount` など）を削除
- バリデーションを強制するイベントリスナーを削除

## インストール方法
1. このリポジトリをクローンまたはダウンロード
2. Chrome を開き、`chrome://extensions/` にアクセス
3. **デベロッパーモード** を有効にする（右上のトグルスイッチ）
4. **パッケージ化されていない拡張機能を読み込む** をクリックし、拡張機能のフォルダを選択
5. 拡張機能が読み込まれ、使用可能になります

## 使い方
1. フォームがあるウェブページを開く
2. Chromeツールバーの拡張機能アイコンをクリック
3. バリデーションが無効になったことを示す確認アラートが表示される
4. クライアントサイドの制限を回避し、フォームを送信可能に

## ファイル構成
- `manifest.json` - 拡張機能の設定ファイル
- `background.js` - 拡張機能のクリックイベントを管理
- `content.js` - ウェブページ上のバリデーションを無効化するスクリプト
- `popup.html` & `popup.js` - 手動実行用のUI
- `icon.png` - 拡張機能のアイコン
